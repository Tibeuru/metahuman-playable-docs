# Animation Retargeting

## Overview

The animation retargeting system is the most sophisticated part of the plugin. It converts standard UE5 Mannequin animations to work with the MetaHuman skeleton.

## Virtual Bone Setup

MetaHuman skeletons lack the IK bones found on the Mannequin (`ik_foot_root`, `ik_foot_l`, `ik_foot_r`). The plugin creates **virtual bones** to bridge this gap:

| Virtual Bone | Replaces        | Parent    | Positioned At |
| ------------ | --------------- | --------- | ------------- |
| `VB root`    | `ik_foot_root`  | `root`    | root          |
| `VB foot_l`  | `ik_foot_l`     | `VB root` | `foot_l`      |
| `VB foot_r`  | `ik_foot_r`     | `VB root` | `foot_r`      |

This approach preserves the original skeleton while enabling full IK compatibility.

## IK Rig Creation

Two IK Rigs are auto-generated:

1. **Source IK Rig** — From the Mannequin mesh (`SKM_Quinn_Simple` or `SKM_Manny_Simple`)
2. **Target IK Rig** — From the MetaHuman's Body mesh

Both are automatically characterized using Unreal's `ApplyAutoGeneratedRetargetDefinition`, which detects bone chains for spine, arms, legs, head, etc.

## Retargeting Pipeline

1. An **IK Retargeter** is created linking the source and target IK Rigs
2. **Batch retargeting** processes all animations from `/Game/Characters/Mannequins/Anims/`
3. Supported animation types:
   - AnimSequence
   - BlendSpace and BlendSpace1D
   - AnimMontage
   - AimOffsetBlendSpace
   - AnimBlueprint

## Asset Naming Convention

Retargeted assets follow this naming pattern:

| Original       | Retargeted                |
| -------------- | ------------------------- |
| `MM_Idle`      | `MM_MHC_Lucy_Idle`        |
| `MF_Jog_Fwd`  | `MF_MHC_Lucy_Jog_Fwd`    |
| `ABP_Unarmed` | `ABP_MHC_Lucy_Unarmed`   |
| `BS_IdleWalk`  | `BS_MHC_Lucy_IdleWalk`    |
| `AO_AimOffset` | `AO_MHC_Lucy_AimOffset`  |

The `MHC_<Name>` identifier is inserted after the type prefix.

## Control Rig Duplication

Three Control Rigs are duplicated and configured for each MetaHuman:

| Source                     | Destination                    | Purpose                  |
| -------------------------- | ------------------------------ | ------------------------ |
| `CR_Mannequin_Body`       | `CR_MHC_<Name>_Body`          | Full body IK             |
| `CR_Mannequin_FootIK`     | `CR_MHC_<Name>_FootIK`        | Foot placement IK        |
| `CR_Mannequin_Procedural` | `CR_MHC_<Name>_Procedural`    | Procedural adjustments   |

Each Control Rig is:

- Reimported with the MetaHuman skeleton (including virtual bones)
- Has its preview mesh updated
- Has curves imported from the skeletal mesh
- Is recompiled with the new hierarchy

The **FootIK** Control Rig also has its bone references remapped from Mannequin IK bones to the virtual bones.

## AnimBlueprint Updates

The retargeted AnimBlueprint has all `ControlRig` node references updated to point to the new MetaHuman-specific Control Rig classes, ensuring animations play correctly with the new skeleton.
